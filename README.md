# 🧩 Escape Room Git – Recuperando o Projeto

## Contexto

Você acabou de entrar em uma empresa e recebeu acesso a um repositório que foi completamente bagunçado pelo desenvolvedor anterior. Link do repositório: https://github.com/profdiegocbcastro/escape_room_git

Antes de sair de férias, ele deixou apenas a seguinte mensagem

> "Se você conseguir recuperar o projeto, ele volta a funcionar."

Seu objetivo é investigar o histórico do Git e restaurar o estado correto do projeto.

# 🎯 Desafios

- O commit "login implementado" deveria estar branch feature-login. Crie a branch e remova apenas esse commit da main
- A hotfix foi feita direto na main, crie uma branch seguindo as boas práticas do gitflow e depois coloque o commit na main. Após crie a release e  tag para produção
- Existe um commit chamado "teste temporario" ele não deveria existir
- Existe um merge chamado 'Merge branch 'feature-temp'', ele foi feito por engano, remova
- Existe um commit com a mensagem "correção final???". Altere a mensagem para "correção final"
- Um arquivo chamado "config.js" foi perdido. Recupere esse arquivo sem reverter todo o commit.


# 📜Regras do jogo
- Você não pode apagar o repositório e começar do zero.
- Envie o Link do seu repositório para demonstrar que fez as atividades
